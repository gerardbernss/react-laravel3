## 2025-05-15 - [Optimization of List Filtering and Static Data Structures]
**Learning:** In React components that handle large datasets, defining static data structures like table columns inside the component body causes unnecessary re-creations on every render, leading to unstable references. Additionally, performing expensive operations (like `new Date()`) inside a `filter` loop within `useMemo` significantly degrades performance as the dataset grows, because the same values are recalculated for every single item.
**Action:** Always move static data structures outside the component render scope. Pre-calculate any loop-invariant values outside of the loop within `useMemo` to ensure O(n) complexity instead of O(n * constant_factor) where the factor is expensive object creation.
