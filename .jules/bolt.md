## 2026-02-05 - Optimizing Admissions List Processing and Data Retrieval
**Learning:** In this application, the Admissions Index page fetches and filters a large list of applicants entirely on the frontend. The original implementation was performing expensive operations like string lower-casing and date parsing inside the filter loop, which caused noticeable lag with many records. Additionally, the backend was fetching all columns from the database, including large text fields not needed for the list view. By hoisting loop-invariant values and using selective column retrieval, we significantly reduced both frontend processing time and backend memory footprint.
**Action:** Always check for redundant calculations inside loops in list-heavy components like Admissions/Index.tsx. Use selective Eloquent select() statements in controllers to minimize payload size for index views.
