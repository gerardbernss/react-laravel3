## 2024-07-26 - Stabilize Static Props Outside Component Scope
**Learning:** A common performance issue in React is defining static data (like a configuration array for a table) inside a component's render scope. This causes the data structure to be recreated on every render, breaking reference equality (`===`). If this data is passed as a prop to a child component (especially one wrapped in `React.memo`), it will trigger an unnecessary re-render even if the data's content is identical.
**Action:** For any static data structure that doesn't depend on component props or state, define it as a `const` at the module level (outside the component function). This creates a single, stable reference that won't change between renders, preventing needless re-renders in downstream components.
